# App NTRL

**Tarea:** Construir una PWA llamada "APP NTRL" para Natural Ropa Deportiva. Es una aplicación interna para gestionar pedidos y producción de ropa deportiva.

IDENTIDAD VISUAL:
- Color principal: #98f909 (verde lima)
- Color secundario: #000000 (negro)
- Color terciario: #FFFFFF (blanco)
- Aplicar estos colores en toda la interfaz: botones principales, encabezados, acentos y elementos destacados
- El login debe tener un diseño impactante con estos colores
- Tipografía moderna y deportiva
- Logo: fondo verde lima #98f909 con ícono y texto en negro, estilo deportivo y minimalista

INFRAESTRUCTURA:
- Servidor local en la red WiFi de la empresa
- PWA que funcione sin internet y se sincronice al reconectar
- Optimizada para tablet y computador

AUTENTICACIÓN:
- Login con usuario y contraseña
- Nombre de la app en pantalla de login: APP NTRL
- 2 roles: Administrador y Trabajador
- Perfil de usuario: nombre, correo, edad, área asignada
- El administrador crea y gestiona usuarios
- Rate limiting en login: máximo 5 intentos cada 15 minutos por IP
- JWT + refresh tokens (PostgreSQL para persistencia + Redis para revocación)

CATÁLOGO DE PRODUCTOS:
- Filtro en cascada: Deporte → Línea → Producto → Género
- Deportes: Ciclismo, Patinaje, BMX, Accesorios, Fútbol, Lifestyle, Atletismo
- Al seleccionar Género (Niño/Hombre/Mujer/Unisex) se despliegan tallas horizontalmente con casilla de cantidad debajo de cada talla
- Tallas niño: T2, T4, T6, T8, T10, T12, T14
- Tallas adulto: T16, TXS, TS, TM, TL, TXL, T2XL, T3XL
- Un pedido puede tener múltiples productos con géneros diferentes

PEDIDOS:
- Número consecutivo automático (#001, #002...)
- Fecha de creación automática
- Fecha de entrega estimada ingresada manualmente por el administrador
- Campos del cliente: nombre, NIT o cédula, dirección, teléfono, correo
- El cliente puede ser persona natural o empresa
- Adjuntar diseño (jpg, png, pdf) visible para todos los trabajadores
- Campo de descripción/observaciones
- Opción de modificar pedido
- Historial de cambios: registrar quién modificó y cuándo, visible para el administrador
- Búsqueda por número de pedido o nombre del cliente

SISTEMA FINANCIERO (solo visible para administrador):
- Por cada producto: tallas horizontales con cantidad debajo, campo de precio unitario ingresado manualmente, subtotal automático = suma de cantidades × precio unitario
- Total pedido = suma de todos los subtotales
- 3 abonos con método de pago cada uno
- Métodos de pago: Efectivo, Transferencia bancaria (bancos: Bancolombia, Nequi, Davivienda, Bold), Link de pago Bold
- Saldo = Total pedido - Abono1 - Abono2 - Abono3 (calculado automáticamente)

ESTADOS DEL PEDIDO (automáticos):
- Pendiente: ningún área ha iniciado
- En proceso: al menos un área activó "estoy trabajando en esto"
- Completado: todas las áreas marcaron listo
- Entregado: administrador lo marca manualmente

ÁREAS DE PRODUCCIÓN:
- Corte
- Diseño: Tarea 1 (Diseñar) y Tarea 2 (Imprimir) — son 2 tareas separadas
- Sublimación
- Ensamble
- Terminados
- Cada área tiene 3 estados: Pendiente, En proceso, Listo
- Cada trabajador tiene área fija asignada
- Notificación al administrador cuando un área marca listo

DASHBOARD DEL ADMINISTRADOR:
- Ventas mensuales
- Ventas por deporte
- Ventas por línea
- Ventas por producto
- Total pedidos por estado
- Total recaudado vs saldo pendiente
- Rendimiento por trabajador (tareas completadas)
- Exportar reportes a PDF y Excel

CLIENTES:
- El cliente puede ser persona natural o empresa
- Campos: nombre, NIT o cédula, dirección, teléfono, correo

HISTORIAL DE CAMBIOS:
- Registrar quién modificó el pedido y cuándo
- Visible para el administrador

SEGURIDAD:
- Script de inicialización seguro para primer admin con bcrypt real
- Backups automáticos diarios desde el primer deploy
- SELECT FOR UPDATE en trigger de recálculo de totales
- Validación de archivos por magic bytes en middleware de upload

- Solo ejecutar código después de aprobación humana

```nginx
# nginx/conf.d/default.conf

upstream app_backend {
    least_conn;
    server app:3000 max_fails=3 fail_timeout=30s;
    keepalive 32;
}

limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/s;
limit_req_zone $binary_remote_addr zone=auth_limit:10m rate=5r/m;

gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_min_length 256;
gzip_types text/plain text/css application/json application/javascript
           application/xml image/svg+xml font/opentype;

client_max_body_size 10M;

# HTTP → HTTPS redirect
server {
    listen 80;
    server_name _;
    location /health { return 200 'OK'; add_header Content-Type text/plain; }
    location / { return 301 https://$host$request_uri; }
}

# HTTPS principal
server {
    listen 443 ssl http2;
    server_name _;

    # SSL
    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;

    # Restricción red local
    allow 192.168.0.0/16;
    allow 10.0.0.0/8;
    allow 172.16.0.0/12;
    allow 127.0.0.1;
    deny all;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Strict-Transport-Security "max-age=31536000" always;

    root /usr/share/nginx/html;
    index index.html;

    # Health check
    location /health {
        allow all;
        access_log off;
        return 200 '{"status":"ok"}';
        add_header Content-Type application/json;
    }

    # Auth — rate limit estricto
    location /api/auth/ {
        limit_req zone=auth_limit burst=3 nodelay;
        proxy_pass http://app_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection "";
    }

    # SSE — sin buffering, timeouts largos
    location /api/notifications/stream {
        proxy_pass http://app_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Connection "";
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
        add_header X-Accel-Buffering "no";
        chunked_transfer_encoding on;
    }

    # API general
    location /api/ {
        limit_req zone=api_limit burst=20 nodelay;
        proxy_pass http://app_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection "";
        proxy_no_cache 1;
        proxy_cache_bypass 1;
    }

    # Assets estáticos — cache 1 año
    location ~* \.(js|css|png|jpg|jpeg|gif|svg|woff|woff2|ttf|ico)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
        try_files $uri =404;
    }

    # Service worker y manifest — cache corto
    location ~* (sw\.js|manifest\.json)$ {
        expires 1h;
        add_header Cache-Control "no-cache, must-revalidate";
    }

    # SPA fallback
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Bloquear archivos sensibles
    location ~ /\.(env|git|htaccess) { deny all; return 404; }
}
```

## 1.6 Dockerfiles

### Server (Multi-stage)
```dockerfile
# server/Dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
RUN apk add --no-cache python3 make g++
COPY package.json package-lock.json* ./
RUN npm ci --only=production

FROM node:20-alpine AS production
WORKDIR /app
RUN addgroup -g 1001 -S appgroup && \
    adduser -S appuser -u 1001 -G appgroup
RUN apk add --no-cache curl tini
COPY --from=deps --chown=appuser:appgroup /app/node_modules ./node_modules
COPY --chown=appuser:appgroup ./src ./src
COPY --chown=appuser:appgroup package.json ./
RUN mkdir -p /app/uploads /app/logs && \
    chown -R appuser:appgroup /app/uploads /app/logs
ENV NODE_ENV=production PORT=3000
VOLUME ["/app/uploads", "/app/logs"]
HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD curl -f http://localhost:3000/api/health || exit 1
EXPOSE 3000
USER appuser
ENTRYPOINT ["tini", "--"]
CMD ["node", "src/server.js"]
```
